## Xcode Signing Error: "Assign a team to the targets" — 解决方案

### 错误信息
```
Use the Signing & Capabilities editor to assign a team to the targets and build a new archive.
```

### 根本原因
`project.yml` 的 signing 配置和 Archive/Release 构建不兼容，导致 Xcode 无法在 Archive 时正确读取团队信息。

### 完整解决方案

**原则：Debug 跳过签名（simulator），Release 开启签名（archive）**

每个 target 都需要在 `project.yml` 中配置：

```yaml
settings:
  base:
    CODE_SIGN_STYLE: Automatic
    DEVELOPMENT_TEAM: 9L6N2ZF26B
    CODE_SIGNING_ALLOWED: NO          # simulator 跳过签名
    CODE_SIGN_ENTITLEMENTS: ...
  configs:
    Debug:
      CODE_SIGNING_ALLOWED: NO        # simulator 构建
    Release:
      CODE_SIGNING_ALLOWED: YES       # archive 构建

# Widget extension 同样处理
CODE_SIGNING_ALLOWED: NO
CODE_SIGN_ENTITLEMENTS: ...
configs:
  Debug:
    CODE_SIGNING_ALLOWED: NO
  Release:
    CODE_SIGNING_ALLOWED: YES
```

### 关键教训
1. **base level 不能有 CODE_SIGNING_ALLOWED=NO**（会阻止 Release/Archive 签名）
2. **必须用 per-config 覆盖**：Debug=NO，Release=YES
3. **Widget 也需要 Release=YES**（否则和主 App 证书不匹配，报 "Not Code Signed" 错误）
4. **必须和成功项目（JustZenGo）对齐**：JustZenGo 没有 CODE_SIGNING_REQUIRED/ALLOWED 在 base level
5. **每次改 project.yml 后必须**：push → MacinCloud pull → XcodeGen → 验证构建

### 验证方法
Archive 前在 MacinCloud 用以下命令验证 Release 配置：
```bash
grep -B3 -A20 'buildConfiguration.*Release' project.pbxproj | grep CODE_SIGNING_ALLOWED
# 应该看到 YES
```
